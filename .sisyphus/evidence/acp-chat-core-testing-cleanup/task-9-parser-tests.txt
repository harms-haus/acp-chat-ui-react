## Test Coverage Areas

### 1. Malformed JSON Handling (5 tests)
- Invalid JSON strings
- Truncated JSON
- JSON with trailing commas
- Empty strings
- Non-version errors in parseEnvelopeSafe

### 2. Missing Required Fields (4 tests)
- Missing version field
- Missing seq field
- Missing timestamp_ms field
- Missing type field

### 3. Invalid Field Types (4 tests)
- Version as string (rejected)
- Seq as string (parsed but type-unsafe)
- Null envelope (TypeError)
- Array instead of object (BridgeVersionError)

### 4. Version Validation Edge Cases (7 tests)
- Version 0 (rejected)
- Version 2 (rejected)
- Negative versions (rejected)
- Very large versions (rejected)
- Floating point versions (rejected)
- NaN, Infinity edge cases

### 5. All 6 Message Type Variants (14 tests)
- **acp_payload**: JSON-RPC request, response, error
- **bridge_status**: All 5 status variants (starting, connected, reconnecting, disconnected, error)
- **stderr**: Empty line, multiline content
- **process_exit**: Code 0, signal only, both code and signal
- **replay_metadata**: Null description, large total_envelopes
- **start_agent**: Minimal config, full config, special characters

### 6. Extra Data Field (4 tests)
- With extra_data
- Without extra_data
- Null extra_data
- Complex nested extra_data

### 7. Boundary and Numeric Edge Cases (5 tests)
- Seq = 0
- Very large seq (MAX_SAFE_INTEGER)
- Timestamp_ms = 0
- Very large timestamp_ms
- Negative seq

### 8. validateEnvelope Function (2 tests)
- Throws for invalid version
- Does not throw for valid version

### 9. createUnsupportedVersionError Function (2 tests)
- Correct structure
- Spread supported versions

### 10. parseEnvelopeSafe Comprehensive Error Handling (4 tests)
- Returns BridgeVersionError for version 0
- Returns BridgeVersionError for version 2
- Throws SyntaxError for invalid JSON (not caught)
- Returns valid envelope for correct input
