## Fixture Format Support

### Captured Format
Location: `fixtures/replay-data/captured/{timestamp}/`
- `replay-events.jsonl` - JSONL format replay events
- `session-data.json` - Session metadata

### Structured Format
Location: `fixtures/replay-data/{demoType}/session-{n}/`
- `replay-events.jsonl` - JSONL format replay events
- `manifest.json` - Manifest with session metadata
- `{sessionId}/session-data.json` - Pre-existing session state

Supported demo types:
- tool-calling
- simple-thought
- permission-request
- tool-calling-thinking
- multi-session
- complex-content
- long-context
- filesystem-test

## Usage Example

```typescript
import {
  loadFixture,
  ReplayRunner,
  assertReplaySuccess,
  assertHasSessionUpdates,
} from '@harms-haus/acp-chat-core/test-utils';
import { ReplayController } from '@harms-haus/acp-chat-core';

// Load a fixture
const fixture = loadFixture('tool-calling/session-1');

// Create and configure runner
const controller = new ReplayController({ bridgeUrl: 'ws://localhost:8080/replay-v2' });
const runner = new ReplayRunner({
  controller,
  fixture,
  replaySpeed: 1.0,
});

// Auto-respond to permission requests
runner.autoRespondToPermissions('approve');

// Execute replay
const outcome = await runner.execute();

// Assert on outcomes
assertReplaySuccess(outcome);
assertHasSessionUpdates(outcome, {
  update: { type: 'agent_thought_chunk' },
}, 3);

// Access statistics
console.log(runner.getStatistics());
// { totalEvents: 50, inboundEvents: 50, outboundEvents: 0, totalTokens: 1234, durationMs: 5000, errorCount: 0 }
```

## Key Features

1. **Fixture Loader**: 
   - Automatically detects captured vs structured format
   - Parses JSONL events efficiently
   - Loads session metadata and pre-existing state
   - Handles both flat and nested directory structures

2. **Replay Runner**:
   - Integrates with ReplayController
   - Records all session updates
   - Tracks permission requests and responses
   - Collects execution statistics
   - Supports configurable replay speed

3. **Assertion Helpers**:
   - Type-safe assertions on replay outcomes
   - Filter session updates by criteria
   - Verify permission request handling
   - Validate statistics
   - Check event sequences
   - Generate human-readable summaries
